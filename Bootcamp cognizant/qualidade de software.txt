*Planejar: 
-Estratégia, Plano de testes, Cenários Básicos 
-Ferramentas Azure, jira, TestLink
-Leitura de requesitos -> Levamento de dúvidas -> ideias de teste -> Estimativa

-Técnicas para testear
-Partição pr equivalência(Equivalence Partition)
-*Análise do valor limite(Boundary value analysis)
-1.Identificar as partições equivalentes e os ranges de valores relacionados
2. Identificar os valores máximos e mínimo(Resultando em partições válidas e inválidas.)
3. Desenvolver as conddições de testes para cada valor limite identificzado

-*Tabela de Decisão(Decision table)
-Ajuda a testar regras de negócios 
-Analisa condições e ações
-*Testes por transição de Estados
-Diagramas

-Algumas narreiras:
-Tempo insuficiente
-Orçamento curto
-falata de capacitação para identificar risco
-Flata de capacitação dos desenvolvedores para exectar testes unitários

-Devemos testar até ter confiança nos requesitos, testar todos os testes previstos
